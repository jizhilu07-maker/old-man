## 4. 当前遇到的问题和解决思路

### 问题1：E2E测试浏览器冲突 ✅已解决
**现象**：Puppeteer测试时报错"The browser is already running..."
**解决方案**：
1. 修复`api-config.js`中的CommonJS导出问题，添加浏览器兼容性检查
2. 为Puppeteer配置独立的用户数据目录，避免多个实例冲突
3. 改进浏览器生命周期管理，确保测试结束后正确清理资源
4. 创建专用E2E测试配置（jest.e2e.config.js），设置串行运行
5. 添加测试失败截图功能，便于调试
**结果**：E2E测试现在可以正常运行，所有3个测试用例全部通过

### 问题2：筛选功能性能优化
**现象**：筛选与搜索组合时可能重复获取数据
**当前状态**：DataService.filterElderlyHomes支持传入已获取的数据
**解决思路**：
- 在handleSearch中先进行文本搜索，再对结果应用筛选
- 避免重复调用getElderlyHomes

### 问题3：测试覆盖率维护
**当前状态**：98.63%覆盖率，需要保持
**解决思路**：
- 每次新增功能都先写测试
- 定期运行覆盖率检查
- 关注未覆盖的代码行

## 5. 重要的文件路径和注意事项

### 主要文件
- `index.html` - 主页面，包含筛选UI控件
- `script.js` - 主JavaScript逻辑，包含筛选功能实现
- `style.css` - 样式文件，包含筛选器样式
- `data/api-config.js` - 数据服务，包含筛选方法
- `tests/unit/dataService.test.js` - 数据服务单元测试（包含筛选测试）

### 筛选相关代码位置
1. **筛选UI控件**：index.html中的filters-section部分
2. **筛选器初始化**：script.js中的initFilters函数
3. **筛选处理**：script.js中的handleSearch函数（第145-168行）
4. **筛选方法**：api-config.js中的filterElderlyHomes方法（第357-394行）
5. **清除筛选**：script.js中的clearFilters函数

### 注意事项
1. **筛选逻辑顺序**：先文本搜索，后应用筛选条件
2. **空值处理**：筛选条件为空时返回所有数据
3. **XSS防护**：所有用户数据都经过escapeHtml处理
4. **响应式设计**：筛选器在小屏幕下适配单列布局
5. **测试驱动**：新增功能前先编写测试
